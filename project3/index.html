Project 3: Image Warping and Mosaicing

Oct 2025 · CS180/280A

Overview

This project explores the process of creating image mosaics by aligning and stitching multiple photographs. The first part covers the manual pipeline: after capturing a set of overlapping images, I define point correspondences, compute the homography transformation that maps one image plane to another, and warp the images into a shared coordinate system. These warped images are then blended to create a seamless mosaic. The second part of the project automates this process. It implements a pipeline to find correspondences automatically, starting with Harris corner detection to identify interest points, followed by feature descriptor extraction and matching. Finally, RANSAC is used to robustly estimate the homography from these matches, allowing for the fully automatic creation of mosaic images.

A.1 · Shoot the Pictures

All photos were taken on my phone at 1×. To approximate a fixed center of projection, I steadied the phone near the lens with my left hand and rotated it about its vertical axis with my right, capturing frames one after another. This kept the camera’s position effectively fixed while I moved through the sequence, producing image pairs that satisfy the homography assumption with roughly 40–70% overlap.

A.2 · Recover Homographies

A homography is a projective transformation that maps points from one image plane to another, assuming both images capture the same planar surface (or were taken by rotating the camera about a fixed center). Mathematically, it is defined as $\mathbf{p}'\,\sim\,H\,\mathbf{p}$, where $H$ is a $3\times 3$ matrix with 8 degrees of freedom (the last entry can be fixed to 1). By selecting at least four pairs of corresponding points — for instance, 8 points from Image A and their matching 8 points from Image B — we can solve for $H$ using a linear least-squares system. Once $H$ is known, it describes how to warp one image so that it aligns with the other. Correspondences are marked with red “x” overlays.

I used eight point correspondences. Each matching pair from the first and second images contributes two linear equations under a 3×3 homography. After fixing the overall scale (setting the bottom‑right entry to 1), the homography has eight free parameters. With eight pairs, the system is overdetermined and I estimate the parameters with least squares, which is far more stable than using only four pairs.

A.3 · Warp the Images

With a homography $H$, I render via inverse warping: for each output pixel, map through $H^{-1}$ to a sub-pixel location in the source and sample there. Canvas bounds come from projecting source corners; out-of-bounds are masked with an alpha channel. Pixel coordinates are treated as centered.

Nearest neighbor. For each back-projected coordinate $(x_s, y_s)$, round to the nearest integer pixel index and copy that value.

Bilinear. For each $(x_s, y_s)$, take the weighted average of the four surrounding pixels using the fractional offsets as weights.

At this resolution, nearest neighbor and bilinear produced similarly smooth, visually reasonable results. Bilinear averages the four neighboring samples, which can soften jagged edges introduced by sub-pixel resampling, while nearest neighbor simply picks the closest pixel and can introduce “stair-step” artifacts—differences that become more apparent on lower-resolution or high-contrast content. In runtime, nearest neighbor was faster: warps that took ~30 s with nearest neighbor took ~50 s with bilinear.

A.4 · Blend the Images into a Mosaic

To create the mosaics, I first determined the size of the output canvas by projecting the corner coordinates of each image through its corresponding homography. This establishes the bounding box that fully contains all warped images. Using these bounds, I warped each input image into the shared coordinate frame with inverse warping, so that every output pixel maps back to its source location. Alongside each warped image, I generated an alpha mask indicating which pixels contained valid content, ensuring that empty or uncovered regions in the canvas were properly ignored.

Each image was warped using bilinear interpolation, which computes smooth pixel values from neighboring samples and avoids aliasing or blockiness along slanted edges. I also applied a geometric offset to translate all warped images into positive coordinate space, allowing them to align correctly within the final mosaic.

Finally, I blended the warped images together using weighted averaging. To compute the blending weights, I applied a distance transform to each alpha mask so that pixels near image centers receive higher weights while those near the borders fade smoothly to zero. This spatial weighting acts as a simple feathering method, softening boundaries and reducing visible seams between overlapping regions. The result is a continuous mosaic that maintains smooth transitions and consistent color across the combined images.

B.1 · Harris Corner Detection

I detect interest points using the Harris corner detector, which identifies locations where the image gradient changes significantly in multiple directions. The Harris response function combines derivatives in both x and y directions to find corners, edges, and flat regions. To select the most distinctive points, I apply Adaptive Non-Maximal Suppression (ANMS), which ensures corners are well-distributed across the image by suppressing weaker responses that are too close to stronger ones.

Concretely, ANMS assigns each detected corner i a suppression radius \(r_i\) defined by the nearest clearly stronger neighbor: \(r_i=\min_j \|x_i-x_j\|\ \text{s.t.}\ f_j>0.9\,f_i\). Intuitively, if a stronger feature lies nearby, \(r_i\) is small (the point is redundant); if not, \(r_i\) is large (the point is distinctive and isolated). The goal is to rank corners by this radius and keep those with the largest \(r_i\), yielding a set of high-quality features that are spatially well-distributed.

B.2 · Feature Descriptor Extraction

After detecting corners with ANMS, I generate feature descriptors to represent each point’s local appearance. My approach is based on a simplified MOPS pipeline. For every interest point, I extract a 40×40 patch centered on it, capturing the surrounding texture and structure. Each 40×40 patch is smoothed and resampled to an 8×8 resolution to create a blurred representation of the local region. The resulting patch is then bias- and gain-normalized for brightness and contrast before being flattened into a 64-element vector. Below are a few examples of the extracted features.

B.3 · Feature Matching

Once descriptors are extracted for both images, the next step is to find reliable correspondences between them. I implemented a feature matching strategy based on the ratio test proposed by Lowe. For each descriptor in the first image, I find its two nearest neighbors in the second image by computing the Euclidean distance between descriptor vectors.

A match is considered reliable only if the distance to the best match is significantly smaller than the distance to the second-best match. Specifically, I keep a pair of features as a match if the ratio of these two distances is below a certain threshold (I used 0.68). This test effectively filters out ambiguous matches where a feature in one image is similarly close to multiple features in the other, which often occurs in repetitive texture regions. B.4 · RANSAC for Robust Homography

While feature matching provides a set of potential correspondences, some of these matches are inevitably incorrect (outliers). To compute a robust homography, I implemented the RANSAC (Random Sample Consensus) algorithm. The process iteratively seeks to find the model that best fits the data in the presence of outliers:

  1. Randomly select four feature pairs from the set of matches.
  2. Compute the exact homography matrix $H$ from these four pairs.
  3. Project all feature points from the first image to the second using $H$ and count the number of inliers—points where the projected location is within a small distance threshold (I used 3 pixels) of the actual matched point.
  4. Repeat this process (steps 1-3) for a fixed number of iterations (I used 2000), keeping track of the homography that produced the largest set of inliers.
  5. Finally, re-compute a more accurate, least-squares homography using all the inliers from the best model found.

This method is highly effective at discarding outliers and finding a homography that aligns the true correspondences. Below is a comparison of the mosaics generated using manually defined points versus the fully automatic pipeline with RANSAC. The automatic approach generally produces more accurate and seamless results, since it relies on precise feature detection and matching rather than manual point selection, which can introduce human error or slight misalignments.
